Node Archive & FTP File Handler.exe 

NODE ARCHIVE & FTP is a software used to handle the file management of remote field sites using the HTI Data Logger or ATR series acoustic equipment that generates *.RAT and *.ATL files. The software monitors a specific folder on the local machine that the HTI equipment has been set up to move completed files of these types into. All other file types are ignored. The software continues to scan this file until a new file is added to the directory, on the addition of a file the NODE ARCHIVE & FTP software moves the file into an Archive directory (and if specified into a redundant Archive directory) then creates a *.ZIP version of the file and retains a copy of it in the Archive directory as well. All Zip files are then copied to a specified FTP host.

NODE ARCHIVE & FTP will not try and move files for the last five minutes of an hour, or for the first five to ten minutes of the hour (based on CPU time). This is to allow the HTI hardware time to move all files at the top of the hour and avoid conflict. During the rest of the hour the software will continue to scan for new files. At any time a user can force a scan or FTP move by selecting the Manual Force FTP button or the Manual Archive Files button.

NODE ARCHIVE & FTP creates an .INI file and a .CSV file in the programs application directory. The .INI file retains the last values of the file directory and FTP credentials used. The .CSV retains a history of *.ZIP files moved to the FTP and the status of their move, if there were errors or not. NODE ARCHIVE & FTP will continue to try and send the file to the FTP host until that file has completed the process with no errors. DO NOT remove or edit the .CSV file or the software will try and move all files up to the FTP again and will slow down the FTP Host.

FTP Connection & Directories (Front Tab)

      DIRECTORY SETUP  

 

Directory To Find *.RAT files:   This is the directory that you specify to find the *.RAT files. *.RAT files are moved to the local machine from the node, this should be the directory those files are being moved into. "Node Archive & FTP" will move these files into the archive directory and this directory should remain empty.

 

Directory To Archive *.RAT files:  This is the directory that "Node Archive & File Manager" will move the *.RAT (and if checked the *.ALT) files into for archiving. This is also the directory "Node Archive & File Manager" will create and store *.ZIP files for each *.RAT and *.ATL included in the directory. This directory is also the one that will be searched for *.ZIP files and then copied to the specified FTP host.

 

(Optional) External Directory To Archive *.RAT files:  If you have an external drive or USB hooked up to the local machine, you can use this directory for a redundant backup location. Otherwise leave blank.

 

 

      FTP SETUP  

These are the Login and File Directory credentials for establishing connection and moving files to an FTP site.

 

Host:  This is the Hosting web address of the FTP client. This should be a web based address of the Hosting provider of the FTP site.

 

UserName:  This is a User Name provided by the local administrator of the FTP site. If you don't know the User Name contact the administrator for the proper login credentials.

 

PassWord:  This is a Password provided by the local administrator of the FTP site and is specific for a given User Name. If you don't know the Password, contact your administrator for the proper login credentials.

 

FTP File Path:   This is the specific file path on the FTP that "Node Archive and FTP" will copy all *.ZIP files. This is a specific file path for each field site; these paths have already been created by your FTP administrator. If you are not sure of the Path contact your administrator. This file path must be an exact match to the ftp, file paths cannot be created through the "Node Archive and FTP" software. Be sure that your file path ends in a backslash (/) otherwise the file path will append the name of the file as a directory and the Files will not be moved up to the FTP. (For example: VAMP2011/USFW_SiteA/)

 

 

Site ID:   The Site ID is a specific identification for the individual sites. Even though a site may have more than one node, the Site ID is referencing the file storage location of the local machine.

 

FTP Timeout (S):   The FTP timeout is not used in this application.

 

Archive and Zip *.ATL Files:   When "Archive and Zip *.ATL Files" is selected "Node Archive and FTP" will move *.ATL files along with the rat files. This includes Archiving them in the Archive and optional Archive directories, along with creating *.ZIP's of each *.ATL and Archiving them along with sending them to the FTP. If the Node is sending *.ATL files to the local machine and you have not selected to archive *.ATL files they will stay in the same directory the node moves them into.

 

Compare file sizes of FTP and Local directory:   When selected the software will compare the file sizes of files on the FTP and those on the Archive Local Directory & if there are discrepancies the software will queue for a replacement of the file on next FTP move session.

      BUTTON COMMANDS:

 

ShutDown & Exit (file pull down):  Will shutdown the software with a prompt asking if you are sure you want to exit the software. You can use Exit from the pull down menu to exit the software and you will not be prompted if you are sure you want to shut down the software.

 

Manual Archive: Will force the software to move files from the Directory to find and into the Archive Directory and also create a *.Zip version of the file.

 

Manual FTP: Will force the software to move files up to the FTP host without waiting for the next auto cycle.

 

FTP Connection & Directories (Second Tab)

 

 

 

 

 

 

Email Recipients:   Enter the Email address of all people that need to be notified of non responsive ping attempts from the node and from the FTP hosting site.

 

Note: That the most common condition of a lack of response from pinging the FTP host would be a lost network connection. This will also prevent Email notifications.

 

Email Recipients:   This is the Email credentials that the software used to send out an email notification. This can only be set in the .INI file after the first time the  software is ran and closed, creating the .INI file.

 

Ping Node 1:   This will turn on or off the ping request for ping Node 1.

Address of Node to Ping:   This is the address (IP) of the Node that the software will ping every five minutes for a response. If no response is received, an Email notification will go out to all Emails listed in the Email Recipients every 15 minutes or until a response is returned.

Node 1 Response:   This is an indicator for the response of the node. If the return is green, then the ping was successful. If the return is red, the ping failed.

 

Ping Node 2:   This will turn on or off the ping request for ping Node 2.

Address of Node to Ping:   This is the address (IP) of the Node that the software will ping every five minutes for a response. If no response is received, an Email notification will go out to all Emails listed in the Email Recipients every 15 minutes or until a response is returned.

Node 2 Response:   This is an indicator for the response of the node. If the return is green, then the ping was successful. If the return is red, the ping failed.

 

Ping FTP Host:   This will turn on or off the ping request for ping FTP Host.

Address of FTP Host to Ping:   This is the address (IP) of the Node that the software will ping every five minutes for a response. If no response is received, an Email notification will go out to all Emails listed in the Email Recipients every 15 minutes or until a response is returned.

FTP Response:   This is an indicator for the response of the FTP Host. If the return is green, then the ping was successful. If the return is red, the ping failed.

 

Time to Next File Size Check:   The amount of time in seconds before the software runs a check to see if the file sizes on the FTP and the local Archive directory are the same. When active this check will run once a day at a time specified in the .INI file. By default the time is 12:30 PM.
